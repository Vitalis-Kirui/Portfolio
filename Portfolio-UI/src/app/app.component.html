My Development Proccess

First of all, I strongly believe in CLEAN code. I hate copying a code even if it's from my own code from another project. I believe in typing every bit of it except some Bootstrap snippets and rare circumstances. One can only achieve a clean code if he/she got a plan before creating a folder in their computer for the project. These are my basics development steps for my projects.

01 : Project Birth

This is the phase that the projects idea kicks in. A lot of this phase happens in mind and a piece of paper. Personally writing something down in paper solve half of the challenge.So I take time and write key components of the idea.

I always make sure of exploiting all the features of the project even those I don't know how to implement yet. Doing this have always prove helpful especially when it comes to generating component structures.

02 : UI Design

Figma it is. After having the entire project skeleton in plain english on a paper, I always head to my favorite design software, Figma.

I take my time here, but it also depends on the size of the project. This very crucial step as it predicts a lot on the number of components you will need, the kind of service the project will need. I believe having the design rigth and realistic if having the real software in advance. I saying so because all that remains after this step is converting the plain english from step one, and using this desgn structure to code.
